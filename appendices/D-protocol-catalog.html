16.10 Theory Groups

Each protocol is assigned to one of five theory groups. The group determines the categorical theory used to represent the protocol’s schema internally and governs how models migrate between protocols.

Group Theory Combination Description
A colimit(ThGraph, ThConstraint, ThMulti) + ThWType Constrained multigraph with W-types. The most general setting for hierarchical, constraint-bearing schemas expressed in JSON or YAML surface syntax.
B colimit(ThHypergraph, ThConstraint) + ThFunctor Hypergraph with functorial structure. Used for tabular and relational schemas where rows, columns, and foreign-key relationships are first-class.
C colimit(ThSimpleGraph, ThConstraint) + ThFlat Simple graph with flat structure. Used for binary wire formats and IDL-defined schemas that lack recursive or dependent types.
D colimit(ThGraph, ThConstraint, ThMulti, ThInterface) + ThWType Typed graph with interfaces and W-types. Used for host-language type systems that support interfaces, generics, and recursive types.
E colimit(ThGraph, ThConstraint, ThMulti) + colimit(ThWType, ThMeta) Multigraph with W-types and meta-level structure. Used for document and markup formats where presentation, content, and schema are interleaved.

The theory group of a protocol is determined by the expressive requirements of its schema language, not by its surface syntax. Two protocols with different surface syntaxes may share a theory group if they require the same categorical structure.

16.11 Cross-Format Compatibility

Conversions between protocols in the same theory group are lossless: both the parse and emit functors factor through the same internal representation, so no information is destroyed in the round-trip. For example, migrating a schema from JSON Schema (Group A) to OpenAPI (Group A) to YAML Schema (Group A) and back yields the original schema up to syntactic normalization.

Conversions between protocols in different theory groups are lossy in general: the target theory may lack the expressive capacity to represent all structures definable in the source theory. The direction and degree of loss depend on the relationship between the two groups.

Source Group Target Group Typical Loss
A B Recursive and W-type structure is flattened; only the relational skeleton is preserved.
A C Constraints and multi-edges are preserved; W-type recursion is unrolled to a fixed depth.
A D Lossless for schemas that fit within a typed-graph model; interface declarations are synthesized.
A E Lossless for schema content; meta-level presentation markup is synthesized with defaults.
D A Interface and generic structure is erased; the remaining graph and W-type content is preserved.
D C Interface and W-type structure is flattened; only the simple-graph skeleton is preserved.
E A Meta-level markup is stripped; schema content is preserved.
B A Hyperedges are decomposed into binary edges; functorial structure is encoded as constraints.
C A Flat structure is embedded; W-type positions are left empty.

When a lossy conversion is required, panproto emits a diagnostic listing which structures were approximated and what information was discarded. Lossless conversions produce no diagnostics.
