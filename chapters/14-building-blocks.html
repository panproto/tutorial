14  The Building-Block Landscape

panproto’s 76 protocol definitions are composed from 27 building-block theories via colimit. Each building block captures a structural distinction that affects migration correctness: whether collections are ordered, whether types form coproducts, whether references are recursive, whether edges are linear. This chapter catalogs all 27 and explains when each is needed.

14.1 The Structure-Validation Boundary

Some schema properties are algebraically natural, expressible as GAT sorts, operations, and equations. Others are irreducibly computational: cardinality bounds, regex patterns, format constraints. The distinction matters because algebraic properties compose cleanly through colimits and migrations, while validation properties must be checked imperatively.

Structural (expressible in GATs):

  • Edge uniqueness: dependent sort Edge(s: Vertex, t: Vertex) in ThSimpleGraph
  • Optionality: section/retraction with equation in ThPartial
  • Recursive types: fold-unfold law in ThRecursion
  • Sum types: variant retraction with equation in ThCoproduct
  • Causality: dependent sort Before(e1, e2) with composition in ThCausal
  • Reflexivity: identity edges with equations in ThReflexiveGraph
  • Symmetry: involution with equations in ThSymmetricGraph

Validation (requires ThConstraint):

  • maxLength, minLength: arithmetic
  • pattern: regex
  • minimum, maximum: ordered field arithmetic
  • format (date-time, email, URI): domain-specific

The mechanism for lifting properties into structure is Cartmell’s property-to-structure move (Cartmell 1986). A conditional property like injectivity (“if f(a) = f(b) then a = b”) cannot be stated as an unconditional GAT equation. But you can add a retraction operation r with the unconditional equation r(f(a)) = a, which encodes injectivity as algebraic structure rather than as a logical property. This is why GATs have dependent sorts.

14.6 Equations as Structural Invariants

Only 6 of the 27 theories have equations. Each has been type-checked against the theory’s sort declarations.

ThRecursion: unfold(fold(v)) = v

fold(v) has sort Mu (since fold: Vertex → Mu). unfold(fold(v)) has sort Vertex (since unfold: Mu → Vertex). The right side v also has sort Vertex. Both sides match.

ThReflexiveGraph: src(id(v)) = v and tgt(id(v)) = v

id(v) has sort Edge (since id: Vertex → Edge). src(id(v)) has sort Vertex (since src: Edge → Vertex). The right side v also has sort Vertex. Both sides match. Same reasoning for tgt.

ThSymmetricGraph: three equations

src(inv(e)) = tgt(e): both sides have sort Vertex. tgt(inv(e)) = src(e): both sides have sort Vertex. inv(inv(e)) = e: both sides have sort Edge.

ThCoproduct: variant_of(injection(v)) = v

injection(v) has sort Vertex (since injection: Variant → Vertex). variant_of(injection(v)) has sort Variant (since variant_of: Vertex → Variant). The right side v also has sort Variant. Both sides match.

ThPartial: defined(witness(d)) = d

witness(d) has sort Vertex (since witness: Defined → Vertex). defined(witness(d)) has sort Defined (since defined: Vertex → Defined). The right side d also has sort Defined. Both sides match.

## The Structure-Validation Boundary

The distinction matters because algebraic properties compose cleanly through colimits and migrations, while validation properties must be checked imperatively.

- Edge uniqueness: dependent sort `Edge(s: Vertex, t: Vertex)` in ThSimpleGraph
- Optionality: section/retraction with equation in ThPartial
- Recursive types: fold-unfold law in ThRecursion
- Sum types: variant retraction with equation in ThCoproduct
- Causality: dependent sort `Before(e1, e2)` with composition in ThCausal
- Reflexivity: identity edges with equations in ThReflexiveGraph
- Symmetry: involution with equations in ThSymmetricGraph

- `maxLength`, `minLength`: arithmetic
- `pattern`: regex

The mechanism for lifting properties into structure is Cartmell's property-to-structure move [@cartmell1986]. But you can add a retraction operation `r` with the unconditional equation `r(f(a)) = a`, which encodes injectivity as algebraic structure rather than as a logical property. ## Equations as Structural Invariants

Only 6 of the 27 theories have equations. Each has been type-checked against the theory's sort declarations.

**ThRecursion**: `unfold(fold(v)) = v`

`fold(v)` has sort Mu (since fold: Vertex → Mu). `unfold(fold(v))` has sort Vertex (since unfold: Mu → Vertex). The right side `v` also has sort Vertex. Both sides match.
